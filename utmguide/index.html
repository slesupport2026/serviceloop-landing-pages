THE LINK STACK RULE:

Every CTA in every email, social post, video description, ad, PDF, or resource points to one of these 6 URLs. If you're creating content and need a link, pick from this list. If none of them fit, the content probably doesn't need a link — or we need to discuss whether we're missing a door.

Content Assets vs. Link Stack

YouTube videos, podcast episodes, social media posts, and the MDE show are content assets — they attract attention and build trust. They are NOT in the link stack. They POINT TO the link stack.

For example, the $1M Service Plan YouTube video would have a description that says:

"Ready to build your plan? Grab the free playbook: serviceloopelectrical.com/playbook/"

The video is the content. The playbook page is the door.

UTM Standards

What UTMs Are (Simple Version)

UTM parameters are tags you add to the end of a URL so you can see WHERE your traffic came from in your analytics. When someone clicks a UTM-tagged link, GHL (and Google Analytics) records which source, medium, and campaign sent them. Without UTMs, all your traffic looks the same — "someone showed up." With UTMs, you know "this person came from Meta, clicked a paid ad, from our June webinar campaign."

The 5 UTM Parameters

ParameterWhat It AnswersRequired?Example
utm_source WHERE did they come from? Yes — Always meta, google, youtube
utm_medium WHAT type of traffic? Yes — Always paid-social, organic, email
utm_campaign WHICH campaign or initiative? Yes — Always pricing-tool-launch-may26
utm_content WHICH specific ad or creative? Recommended for ads video-edwin-testimonial
utm_term WHICH keyword? (Google Ads only) Optional electrician-pricing

Formatting Rules (Non-Negotiable)

RuleRightWrong
Always lowercasemetaMeta, META, Facebook
Use hyphens, never spaces or underscorespaid-socialpaid social, paid_social
No special characterswebinar-june26-claywebinar_june'26_(clay)
Keep values short but clearpricing-tool-may26the-pricing-tool-campaign-for-may-2026
Include month+year in campaignswebinar-jul26webinar-summer
Platform = brand name, not productmetafacebook, instagram, fb
WHY THIS MATTERS:

If Forrest tags an ad with utm_source=Facebook and Luigi tags one with utm_source=meta and Clay tags a post with utm_source=fb, your GHL reports show three separate sources that are all the same thing. You can never combine them cleanly. Pick one convention. Use it forever. This is it.

Where to Set UTMs (By Tool)

Meta Ads Manager (Forrest / Ads Team)

Where to enter UTMs:

  1. Go to your ad set or ad level
  2. Scroll to Tracking section
  3. Click Build a URL parameter
  4. Enter each UTM value individually OR paste the full parameter string into "URL Parameters"

Parameter string format:

utm_source=meta&utm_medium=paid-social&utm_campaign=pricing-tool-may26&utm_content=video-edwin-testimonial

Paste this into the "URL Parameters" field. Meta appends it to your destination URL automatically. You do NOT put it in the website URL field.

Vista Social (Scheduled Organic Posts)

Where to enter UTMs:

  1. When composing a post, add your link
  2. Click the link settings (gear icon near the URL)
  3. Vista Social has a built-in UTM builder — fill in each field
  4. You can also set default UTMs in Settings → UTM Tracking so every link auto-tags

Recommended defaults for Vista Social:

utm_source=vista-social / utm_medium=organic-social / campaign = per post

GHL Email Sequences (Luigi / Clay)

Where to enter UTMs:

When adding a link inside an email in GHL, use the full tagged URL from the UTM Builder above. GHL does not auto-tag outbound links — you have to include the UTMs in the URL itself.

Example for AM Morning Coffee Day 12:

https://serviceloop-landing-pages.netlify.app/playbook/?utm_source=email&utm_medium=email-am&utm_campaign=morning-coffee-day12

Example for PM ACDC Chapter 3 Day 2:

https://serviceloop-landing-pages.netlify.app/webinar/?utm_source=email&utm_medium=email-pm&utm_campaign=acdc-ch3-day2

YouTube Descriptions (Content Team)

Where to enter UTMs:

Paste the full tagged URL directly into video descriptions, pinned comments, or community posts.

Example for MDE Episode:

https://serviceloop-landing-pages.netlify.app/sle-pricetool/?utm_source=youtube&utm_medium=organic-video&utm_campaign=mde-ep47-pricing

GHL Receiving UTMs (Luigi)

How GHL captures incoming UTMs:

When a prospect clicks a UTM-tagged link and lands on a landing page, the form JavaScript grabs the UTM values from the URL and sends them as hidden fields in the webhook payload. GHL stores them in custom fields on the contact record.

Required GHL Custom Fields:

utm_source, utm_medium, utm_campaign, utm_content, utm_term

These are already in the GHL Workflow Blueprint checklist. The landing page JavaScript reads URL params and passes them through the webhook. I can update the form code to do this once Luigi provides the webhook URLs.

Short Links — Self-Hosted URL Shortener

Hide tracking params behind clean, branded short URLs. No Bitly. Powered by Netlify _redirects — server-side, instant, zero JavaScript.

How It Works

Instead of sharing go.serviceloopelectrical.com/sle-pricetool/?utm_source=meta&utm_medium=paid-social&utm_campaign=...

You share go.serviceloopelectrical.com/r/pt-meta-edwin

Netlify intercepts /r/pt-meta-edwin, does a server-side 302 redirect to the full UTM URL. The visitor never sees the tracking parameters. ~70% shorter. Branded. Free.
